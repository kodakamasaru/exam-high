# 採用試験 — 全体概要

## 目的
- シニアエンジニアとしての採用可否判断（足切り）
- 給与レンジの判定材料取得（スコア分布）

## 基本方針
- AI使用推奨（禁止しない）
- 評価は100%自動（E2E足切り + 静的解析スコア + 動的テストスコア）
- 面接ではADRの深掘りのみ行う
- アーキテクチャは自由（レイヤー型でもモジュール型でも可）
- **「良い設計を直接評価しない。構造違反・品質劣化を検出する」**

## 試験形式
| 項目 | 値 |
|---|---|
| 対象 | シニア（アーキテクト）レベル |
| 期間 | 72時間（持ち帰り） |
| 提出 | Git |
| 環境 | Docker + docker-compose |
| テーマ | 予約管理システム |
| バックエンド | Hono (TypeScript)。構造自由 |
| フロントエンド | Next.js |
| DB | PostgreSQL |
| ORM | 自由（採点対象外） |
| FW | バックエンドHono指定。フロントNext.js指定 |

## タスクタイプ
**「ブラックボックス採点 + 自由設計」方式**
- ゼロから実装。v1/v2の概念なし
- API仕様は完全固定。内部設計は完全自由

### 受験者に提供するもの
- API仕様書（OpenAPI形式。エンドポイント・リクエスト/レスポンス形式を定義）
- APIテスト（supertest。セルフチェック用）
- E2Eテスト（Playwright。セルフチェック用）
- data-testid仕様書（フロントエンドで付与すべきtestidの一覧）
- Docker基盤（PostgreSQL, Redis等のインフラのみ）
- ADRテンプレート（ADR-000 Architecture Overview は必須提出）
- README（起動方法、提出方法）

### 採点側のみ保持（受験者には非公開）
- 負荷テストスクリプト（k6）
- セキュリティスキャン設定（Bearer）
- 静的解析ルール（dependency-cruiser, ESLint設定, etc.）
- シードデータ投入スクリプト（大量データ）
- 採点パイプライン（Makefile + score.js）

**方針**: 仕様確認用のテスト（E2E・API）は提供する。採点ロジック（静的解析・負荷テスト・セキュリティ・配点）は非公開。
テストを見てもスコアに影響する採点基準は一切わからない。

### 受験者が自由に設計・実装するもの
- 内部アーキテクチャ（レイヤー型/モジュール型/その他、全て可）
- DBスキーマ
- ビジネスロジック
- フロントエンド実装

## 配点方針
- **アーキテクチャ・型安全性（保守性・可読性）= 厚め**
- **セキュリティ・パフォーマンス = 薄め**（後から巻き返せるため）

## アーキテクチャ採点の仕組み
- 受験者はADR-000でディレクトリ構造（モジュール名・パス・責務）を文書化（面接用ドキュメントとして自然）
- 採点側はADR-000からモジュール境界を読み取り、dependency-cruiserで依存違反を自動検出
- 受験者は採点方法を知らない
